Conceptos básicos

Algoritmo Secuencia de instrucciones no ambiguas para resolver un problema dentro de un marco de tiempo finito
Entrada El dato (o datos) que recibe el algoritmo. Puede ser el tamaño de un arreglo, la cantidad de palabras, etc. Es representado por n
Complejidad temporal Tiempo que requiere un algoritmo de inicio a fin. Se puede medir en segundos o milisegundos
Notacion O grande Complejidad temporal maxima, representada como O(n)

¿Qué es Fuerza Bruta?

La fuerza bruta (Brute Force) es el método más sencillo de implementar para diseñar algoritmos para solucionar problemas. Se basa en revisar cada posible opción dentro de un problema hasta encontrar con la solución.

Aunque es el método más fácil de implementar, también es el menos efectivo, debido a que debe revisar elemento por elemento, por lo que su complejidad tiende a ser lineal o exponencial. Debido a esto, la fuerza bruta solo es viable para problemas pequeños.

Ejemplos de Problemas Resueltos Por Fuerza Bruta

Busqueda de patrones

Conocido como Busqueda Bruta (Brute Search), este algoritmo recibe una cadena de caracteres, conocida como el Haystack, y una palabra, conocida como el Needle. El objetivo es encontrar la palabra dada dentro de la cadena.

Como ya se mencionó, por medio de Fuerza Bruta se debe analizar posición por posición no solo el Haystack, pero también el Needle, lo que causa que este algoritmo tenga una complejidad de O(n+m), donde n es el tamaño del Haystack y m el tamaño del Needle

Este tipo de problemas aparecen en Data Mining, programas de procesamiento de texto y en el análisis de secuencias de ADN

Distancia más corta entre dos puntos

Dado una lista de coordenadas en un plano, hallar los dos puntos con menor distancia entre sí.

Por fuerza bruta, el algoritmo compara todos los puntos dados con todos, por lo que su complejidad es de O(n²) donde n es el número de puntos. Debido a esta complejidad, solo es viable para problemas con pocos puntos. Alternativas para problemas más grandes incluyen los algoritmos de Dijsktra y Bellman-Ford

Escenarios donde se debe hallar la distancia mínima entre dos puntos incluyen casos de mapeo. Planeación de rutas y optimización de distancias y costos
